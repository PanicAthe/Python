# 6_mnist_nn 개념/공부 요약

## 1. 이 노트북에서 한 일 (큰 흐름)
- MNIST 손글씨 데이터셋을 불러와 데이터 크기와 샘플 이미지를 확인함
- 레이블 분포를 확인하고, 타깃을 one-hot encoding으로 변환함
- 픽셀값 정규화(0~255 -> 0~1) 후 이미지를 784차원 벡터로 reshape함
- Dense 기반 다층 신경망(100-50-10)을 구성하고 학습함
- 테스트셋 정확도를 평가하고, 앞 10개 샘플 예측 결과를 정답과 비교함

## 2. 핵심 개념 정리

### 2-1. 이미지 입력 전처리 (정규화 + 펼치기)
- 원본 입력은 `(28, 28)`의 2차원 이미지임
- Dense 층에 넣기 위해 각 이미지를 `28*28=784` 길이의 1차원 벡터로 변환함
- 정규화(`x/255`)로 입력 스케일을 맞춰 학습 안정성과 수렴 속도를 개선함
- shape 변화
  - 변환 전: `x_train (60000, 28, 28)`, `x_test (10000, 28, 28)`
  - 변환 후: `x_train (60000, 784)`, `x_test (10000, 784)`

### 2-2. 다중분류 출력 형식 (one-hot + softmax)
- 숫자 클래스는 `0~9`의 10개이므로 다중분류 문제임
- `to_categorical(..., num_classes=10)`으로 레이블을 one-hot 벡터로 바꿈
- 출력층 `Dense(10, activation='softmax')`가 각 클래스 확률을 출력함
- 손실함수로 `categorical_crossentropy`를 사용해 one-hot 타깃과 맞춤

### 2-3. MLP 구조와 파라미터 해석
- 모델 구조
  - `Dense(100, activation='sigmoid', input_shape=(784,))`
  - `Dense(50, activation='relu')`
  - `Dense(10, activation='softmax')`
- 파라미터 수 계산
  - 1층: `(784*100)+100 = 78,500`
  - 2층: `(100*50)+50 = 5,050`
  - 3층: `(50*10)+10 = 510`
  - 총합: `84,060`

## 3. 코드 흐름 요약

### 3-1. 데이터 준비
- 데이터 로드: `tf.keras.datasets.mnist.load_data()`
- 샘플 시각화: `matplotlib`로 첫 25개 이미지 확인
- 레이블 분포 확인: `np.unique(y_train, return_counts=True)`
- 타깃 one-hot 변환: `y_train_oh`, `y_test_oh`

### 3-2. 모델 정의/학습
- 모델: `Sequential` Dense 3층(100-50-10)
- 컴파일: `loss='categorical_crossentropy'`, `metrics=['accuracy']`
- 학습: `epochs=10`, 기본 `batch_size=32` (60000/32=1875 step)
- 학습 로그에서 epoch가 진행될수록 정확도 상승, 손실 감소를 확인함

### 3-3. 평가/예측
- 테스트 평가: `model.evaluate(x_test, y_test_oh)`
- 예측: `model.predict(x_test[0:10])`
- 클래스 결정: `np.argmax(pred, axis=1)`
- 정답 비교: `y_test[0:10]`와 대조

## 4. 실행 결과 핵심
- one-hot shape: `y_train_oh (60000, 10)`
- 모델 총 파라미터: `84,060`
- 마지막 epoch(10) 학습 지표: `accuracy: 0.9864`, `loss: 0.0444`
- 테스트셋 지표: `accuracy: 0.9777`, `loss: 0.07777409255504608`
- 첫 10개 예측 클래스: `[7 2 1 0 4 1 4 9 5 9]`
- 첫 10개 실제 클래스: `[7 2 1 0 4 1 4 9 5 9]`

해석
- 단순 MLP 구조만으로도 MNIST에서 약 `97.8%`의 높은 테스트 정확도를 달성함
- 앞 10개 샘플은 예측과 정답이 모두 일치해 추론 파이프라인이 정상 동작함
- 다만 코드상 출력된 경고처럼 `Sequential`에서는 `input_shape`를 첫 Dense에 직접 넣기보다 `Input(shape=(784,))`를 분리하는 방식이 권장됨

## 5. 공부 포인트
- Dense 모델에서 입력 shape 변환(2D 이미지 -> 1D 벡터)의 이유를 명확히 이해하기
- one-hot 타깃과 `categorical_crossentropy` 조합이 왜 맞는지 정리하기
- 파라미터 수를 층별로 직접 계산해 모델 크기를 감각적으로 익히기
- 학습 정확도(0.9864)와 테스트 정확도(0.9777) 차이로 일반화 성능을 해석해보기
